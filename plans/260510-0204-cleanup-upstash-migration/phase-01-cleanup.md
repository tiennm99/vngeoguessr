# Phase 1: Cleanup

## Overview

Single-pass cleanup of Upstash migration scaffolding. No code logic changes, only file/dependency removal and plan-status update.

## Related Code Files

- Modify: `package.json` (remove `redis` dep) — **highlight to user** per File Modification Policy
- Modify: `package-lock.json` (regenerated by `npm uninstall`) — **highlight to user**
- Delete: `scripts/migrate-upstash.js`
- Delete: `.env.migrate` (local only, gitignored)
- Update: `plans/260509-2234-upstash-db-migration-with-prefix/phase-03-cutover-and-cleanup.md` (mark complete via `ck plan check 3`)

## Implementation Steps

1. **User runs**: `npm uninstall redis`
   - Verify: `npm ls redis` returns empty.
   - This regenerates `package-lock.json`.
2. **Agent deletes**: `scripts/migrate-upstash.js`.
   - Recoverable from git history (commit `53ee795`) if ever needed.
   - The directory `scripts/` will be left empty; remove the directory too.
3. **Agent deletes**: local `.env.migrate`.
   - Contains old DB credentials no longer needed; sensitive even if gitignored.
4. **Agent runs**: `cd plans/260509-2234-upstash-db-migration-with-prefix && ck plan check 3` to mark Phase 3 of the migration plan complete.
5. **Agent runs**: `npm run build` to confirm nothing else depends on `redis`.
6. **Agent commits + pushes** the deletion changes (single commit).

## What stays untouched

- Vercel env vars `REDIS_URL`, `KV_URL`, `KV_REST_API_READ_ONLY_TOKEN` — auto-provisioned by the Vercel Marketplace Upstash integration. Removing causes drift.
- `src/lib/mapillary.js` console.error lines — kept-on-purpose error observability, not temporary instrumentation.
- The migration plan files themselves (kept as record).

## Success Criteria

- [ ] `npm ls redis` empty
- [ ] `scripts/migrate-upstash.js` not present
- [ ] `.env.migrate` not present locally
- [ ] `npm run build` passes
- [ ] Original migration plan Phase 3 marked completed
- [ ] Single commit pushed to main

## Risk Assessment

- **Risk**: deleting the migration script loses the bridge if a re-migration becomes necessary. **Mitigation**: git history preserves it (commit `53ee795`). Restoration is `git checkout 53ee795 -- scripts/migrate-upstash.js`.
- **Risk**: removing `redis` dep breaks something we missed. **Mitigation**: `npm run build` step after uninstall surfaces any stray imports. Grep already confirmed no `from 'redis'` or `require('redis')` references in `src/`.
- **Risk**: `.env.migrate` deletion is irreversible. **Mitigation**: file just contains creds copy-pastable from Vercel dashboard if ever needed; no original-source data.

## Out of Scope

- Removing Vercel auto-provisioned env vars.
- Refactoring observability in `mapillary.js`.
- Any change to the new Upstash integration or its credentials.
